The OSI (Open Systems Interconnection) Model

The OSI Model is a conceptual model used to describe the functions and layers needed for system communications. The model was conceived by the Open Systems Interconnection project at the International Organization for Standardization (ISO) ref ISO/IEC 7498-1.

The OSI model describes 7 layers that can be used for communications; some of the layers may be omitted in some cases.  We use the model to describe the process of network continuations form the hardware to the application and receiving and transmitting the data.
The 7 layers are;
  1. Physical layer, describes the physical connection to the system used to transmit the data to the next node.  This would be the cables (Fiber optic, 10baseT, etc) and associated hardware.
  2. Data link layer, is responsible for node to node transmission.  This layer ensures  reliable transmission of data over layer1. For example Ethernet, Token Ring, Frame relay and ATM are Layer 2 protocols.
  3. Network layer, provides routing, congestion control and transport of data over the network. The network layer provides both connectionless and connection-oriented services.  The IP protocol is an example of layer 3.
  4. Transport layer, provides overall end-to-end validation and integrity of the data as well as dividing it into manageable chunks.  This layer is the last layer responsible for the transport of the data over the network.  TCP, and UDP are examples of layer 4. 
  5. Session Layer, permits two parties to hold ongoing communications called a session across a network. The applications on either end of the session can exchange data or send packets to another for as long as the session lasts.  In reality, this layer is often not used or is incorporated into layer 4.
  6. Presentation Layer, provides a method of translating between the application and network formats. Transforming the data into a form that the application and network layers will accept.  This layer formats and/or encrypts the data being sent across the network.
  7. Application layer, is the top layer of the OSI model. It provides a set of interfaces for
    applications to obtain access to the networked.  This would include HTTP, FTP, SNMP, and any application that you would use to access network services.

What is layer 8 ?

Layer 8 refers to services/applications that have been embedded into the transport layer7.  For instance a java toolbar that has been embedded into the HTTP stream.  At this time layer 8 is not an official layer although it is used by many network venders to describe services and offerings.
