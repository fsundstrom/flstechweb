High Availability Linux Systems and Services

The key to providing any network service is up time.  Whether you are dealing with files at home or a large important web site you want the expected services to be available.  This is why Cloud services are so popular,  they do the work to make sure that the files and services are available all the time.  We will go over the basics here knowing that the only sure way is complete site redundancy.

Mirroring Data

Whichever solution you choose you need to make sure that the same user data is available on all servers.  This is true for web sites, file servers and can even be helpful for services like proxy servers.

Active/Standby solutions
Simple easy to set up redundancy is active/standby as you have no load balancing or state issues.  This also means that you have twice the hardware without increasing capacity. 

Interface Bonding:

In Linux you can set up 2 network interfaces so if one goes down the other one becomes active with the same settings (simple hardware redundancy).  To do this you create a bonded virtual interface and assign 2 hardware interfaces to the new bonded interface.  You then set up all the parameters that you would normally have on the Ethernet interfaces to the bonded interface such as IP address, default gateway, etc.

The best way to do this is the have each of the interfaces going to a separate switch so you don't only have redundant Ethernet interfaces you also will fail over in case a switch goes down.  In many cases this goes all the way to the ISPs so you end up with 2 separate networks.  You must be careful to make sure that you have fail over when upstream systems go down.

IP fail over with a Virtual IP address (VIP)

One of the simplest solutions is to use the keepalived package.  This package has built in watchdogs that know the state of the other system and can fail over in case of high load or the system going down.  It also can start or stop services on the active or standby system as well as changing the prompt to indicate witch is active.

Clustering

In some cases clustering makes more sense.  Although harder to set up clustering can be setup for active/standby solutions. 

Active/Active Solutions

Although is does no always make sense active/active architectures have a major advantage in that all the hardware is used.  This means that you gain capacity and at the same time you have redundancy.
