DNS and name resolution

Computers want to use numbers and people like to use names.  So we have name resolution to associate IP address to names.  In most cases this is DNS (Domain Name System).  Their are other services like WINS although DNS is what most computers and networks use today. 

Types of DNS servers

Root servers

The DNS namespace is set up in levels where the top level root servers contain a list of registered domain names and the authoritative name servers associated with the domain.  This list is published by the interNIC and is a very short list 13 servers (in reality it is many more as each IP has many servers associated with it).

Master Servers (Primary)

Master servers contain the DNS zone files for a domain, sub-domain or block of IP address.  It does not need to make quires and is authoritative for the zones.  A master server can also be a caching server or a slave server for other domains.

Slave Server (Secondary)

A slave server can be authoritative. It just gets its zone files from the master server.  It uses the serial number on the zone file and if the master has a higher number.  The slave server request a zone transfer. It does not send each request to the master and just gets the whole zone record when outdated.  In many cases the master servers just answer request form the slaves and do not handle internet traffic.

Caching servers (Resolver)

Caching servers have a hint list of the root servers and respond to queries.  When it receives DNS query it, sends a request to one of the root servers, that then responds with the authoritative DNS server/s for that domain. The caching server then queries that server and gets a response or gets sent to the authoritative server for a subdomaion.  This keeps going down the chain until it gets a response for the query.  The caching server now sends back the answer to the host making the original query and caches the information, so next time it will just send the response form the information in the cache.  Each record has a timeout that tells the caching server how long to keep the record in cache.

Forwarding server (Proxy)

A forwarding server just sends the request up to another server.  When it get the response it caches the result and sends it back to the requester. 


DNS queries

You can send a standard DNS query, or DNSSEC for secure DNS queries and zone transfers.  The most common record types are A (Address record) where you give a system name and get an IPv4 address back.  AAAA (Ipv6 address record) in this case you get an IPv6 address if available.  MX (Mail exchange record) returns a list of mail servers.  PTR (Pointer record) is used to resolve an IP to a hostname.
