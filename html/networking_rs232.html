RS232 is the most common form of communications used for console access to equipment.  

RS232 is one of the first methods used for terminal access to computer systems and has been in use since the 70's.  The first computer systems did not have network access as we know it today and terminals were hard wired to the servers.  Today we use RS232 for console access to systems before we have network access or if somthing happens on the network to prevent accessing the system.

RS232 requires 3 wires to work the rest of the wires are used for signaling if the systems are ready to send or receive data.  The 3 connections needed are ground, send and receive.  You can use hardware flow control witch requires at least 2 more connections (ether DSR/DTR or RTS/CTS).  Normally RTS/CTS is used for flow control and DSR/DTR is used to let the other system know that the equipment is ready for communications.   You can use software flow control where the systems send Xon and Xoff over the wire.

We also need to know when to use a null modem cable.  A null modem cable lets us connect 2 similar hardware types together (i.e. connecting 2 laptops together).  The 2 types of equipment are DTE (data terminal equipment) this would include terminals (we don't see them anymore) and computers.  The other type of equipment is DCE (data communications equipment). 

The next most important thing is the settings on the 2 systems.  The settings are baud (connection speed), Data bits (the number of data bits most common is 8) parity (normally set to odd or none ), stop bits (Normally set to 1 ), and flow control(normally set to CTS/RTS or sometimes software Xon/Xoff).  common speed settings are 110, 150, 300, 600, 1200, 2400, 4800, 9600, 19200, 38400, 56000, 57600 and 115200.

Standard connectors are DB9 DB25 and RJ45 the following are the common pinouts.  Cisco uses the Yost standard, PCs normally have DB9,  and IBM BC uses the EIA standard.
