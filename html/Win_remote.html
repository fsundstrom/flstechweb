Windows Remote Desktop and Terminal Services

Windows Terminal Services gives users access to remote desktops on other computers.  Windows servers can run multiple desktops for multiple users simultaneously.  You local desktop PC (Win7 Win8 ...) can only support one desktop session. 

To run Windows Terminal servers on your computer.  Run "Remote Desktop Connection" under "accessories", this runs "C:\Windows\System32\mstsc.exe".  You then give it the name or IP of the remote host.  By default it uses the mouse, keyboard, and sound card on the local system.  Under "Options" you can do things like; mount a local drive, change sound card settings, optimize for line speeds, etc. 

You need to enable remote access for Terminal Services to access to the remote system.  This is under "System Properties", go to "remote" tab and enable Remote Desktop.  Now you can use your computer from another Windows system.  Terminal Services uses TCP port 3389, so you need to make sure any firewalls between you and the remote system have this port open.

This is one of the most common ways to administrator remote systems including windows servers.  You can also set up windows servers so each user has their own desktop.  Windows servers allow 2 sessions by default and need to be licensed for additional sessions.
