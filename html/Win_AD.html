Windows Active Directory

Microsoft's Active Directory service or AD is based on LDAP (Lightweight Directory Access Protocol).  So basically it is just a directory of users and computers.  Each record has specific information about that user or system. AD also uses Microsoft's own version of Kerberos for user authentication and authorization (i.e. user passwords and permissions).  As well as DNS for resolving system names.

The main use of active directory is to manage users and or systems in an organization.  To use active directory you need to install it on you windows server (or in most cases more then one windows server for redundancy).  An active directory server is referred to as a domain controller as it decides the polices for systems and users in a domain.

The most basic use if AD is to manage users passwords and policies.  In this way you can enforce password polices like changing the password every 4 months or password types.  Administrators can also lock out users or change their password if a user has issues.  The other main use of AD is to enforce data or system access polices for users and systems.

The Active Directory structure has a number of levels, forest, tree, domain and organizational unit.  These are the logical divisions of an Active Directory network.  A forest consists of a number of trees.  Trees consists of one or more domains in the same namespace (i.e. datacenter.mydomain.com, offices.mydomain.com ...).  Each domain shares one database and consists of objects (users, computers, printers ...) that can be grouped into organizational units.  The OU (Organizational Unit) is the recommended level at which to apply group policies.  You can use group policies at the domain level but this leads to a much more complex environment as each domain requires its own domain controllers.
